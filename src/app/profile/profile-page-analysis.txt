📱 سناریو و قابلیت‌های صفحه پروفایل (Profile Page) - Quiz of the Kings

🎯 هدف کلی صفحه:
صفحه Profile امکان مشاهده و ویرایش اطلاعات شخصی کاربر، مشاهده آمار و مدیریت پروفایل را فراهم می‌کند.

🧩 بخش‌های موجود و قابلیت‌های مورد نیاز:

1️⃣ بخش آواتار و اطلاعات اصلی
وضعیت فعلی: UI کامل با قابلیت ویرایش
قابلیت‌های موجود:
- ✅ نمایش آواتار کاربر (فعلاً از Dicebear API برای تست)
- ✅ نشانگر وضعیت آنلاین/آفلاین
- ✅ ویرایش نام با اعتبارسنجی کامل
- ✅ ویرایش یوزرنیم با اعتبارسنجی کامل
- ✅ نمایش سطح و عنوان کاربر
- 📝 **استراتژی آواتار**:
  - **تولید**: استفاده از آواتار تلگرام کاربر
  - **تست**: استفاده از Dicebear API (فعلی)
  - **پیش‌فرض**: تصویر ثابت برای کاربران بدون آواتار

اعتبارسنجی‌های موجود:
- نام: حداقل 2، حداکثر 30 کاراکتر، نمی‌تواند خالی باشد
- یوزرنیم: حداقل 3، حداکثر 30 کاراکتر، فقط حروف انگلیسی، اعداد و _

2️⃣ بخش آمار کاربر
وضعیت فعلی: نمایش داده‌های استاتیک
قابلیت‌های موجود:
- ✅ نمایش امتیاز کل
- ✅ نمایش تعداد سکه‌ها
- ✅ نمایش تعداد بازی‌های انجام شده
- ✅ فرمت فارسی اعداد

3️⃣ سیستم ویرایش اطلاعات
وضعیت فعلی: کاملاً عملکردی
قابلیت‌های موجود:
- ✅ حالت ویرایش inline برای نام و یوزرنیم
- ✅ دکمه‌های ذخیره و لغو
- ✅ نمایش خطاهای اعتبارسنجی
- ✅ مدیریت state محلی
- ✅ UI/UX عالی برای ویرایش

4️⃣ ناوبری و کنترل
وضعیت فعلی: عملکردی
قابلیت‌های موجود:
- ✅ دکمه بازگشت به صفحه اصلی
- ✅ استفاده از useDelayedAction
- ✅ مدیریت back button تلگرام

🎮 سناریوهای کاربری:

سناریو 1: مشاهده پروفایل
1. کاربر وارد صفحه پروفایل می‌شود
2. اطلاعات شخصی و آمار نمایش داده می‌شود
3. کاربر می‌تواند وضعیت فعلی خود را مشاهده کند

سناریو 2: ویرایش نام
1. کاربر روی آیکون ویرایش کنار نام کلیک می‌کند
2. فیلد input نمایش داده می‌شود
3. کاربر نام جدید را وارد می‌کند
4. سیستم اعتبارسنجی انجام می‌دهد
5. در صورت صحیح بودن، نام ذخیره می‌شود

سناریو 3: ویرایش یوزرنیم
1. مشابه سناریو ویرایش نام
2. اعتبارسنجی خاص یوزرنیم (فقط حروف انگلیسی، اعداد و _)
3. بررسی یکتا بودن یوزرنیم (نیاز به API)

سناریو 4: خطا در ویرایش
1. کاربر اطلاعات نامعتبر وارد می‌کند
2. پیام خطای مناسب نمایش داده می‌شود
3. امکان تصحیح یا لغو ویرایش
